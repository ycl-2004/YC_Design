# Scene: 教学深度页（真正教懂零基础）

> 当目标是「让完全不懂的人彻底学会」，而不只是「科普/介绍让人有个印象」时用本场景。
> **本场景是 `scene-tutorial.md` 的教学法升级层**：先完全继承它的视觉规范（色板/字体/布局/装饰/动效/响应式/11 种布局模式），再叠加下面的教学法要求。模板仍用 `assets/template-tutorial.html`。

---

## 触发条件（与"科普/介绍"的区别）

| | 科普/介绍（scene-tutorial 默认） | 教学深度（本场景） |
|---|---|---|
| 目标 | 祛魅、有个印象、产生兴趣 | 彻底学会、能自己推理新情况 |
| 深度 | 每个概念一两句直觉 | 每个概念走完整四拍 |
| 互动 | 基本只读 | 有检查理解、回顾 |
| 触发词 | 介绍、科普、了解一下、是什么 | **教程、讲透、零基础、系统学、课程、teach、lesson、教学 slide** |

拿不准时问一句：用户是想"让人感兴趣"还是"让人学会"？学会 → 本场景。

## 🎓 教学弧线（section 顺序模板）

按这个顺序排 section，每屏布局仍要不同（遵守 scene-tutorial）：

1. **Hook / 承诺** — 一句话说清"跟我走一遍就懂"，并预告将贯穿的那个例子；列 2-3 条"学完你能做到什么"。
2. **引入贯穿例子** — 抛出一个**从头跟到尾的具体例子**（见下方铁律②），给最小的真实数据。
3. **逐个教概念** — 每个新概念走**四拍**（见铁律①），都挂回贯穿例子。
4. **讲机制** — 回答"它到底怎么work/越练越准的"。别只给静态结论，要给**过程**。
5. **一般化** — 把这一个例子推广到更大图景（分类、家族、其它任务），每个抽象点配真实任务。
6. **检查理解** — 至少一处 mini-quiz / reveal-q，让学习者主动回忆、自测。
7. **边界与坑** — 常见误区、易错点（如过拟合），用同一个贯穿例子讲最省力。
8. **小结 + 术语表** — 一页 recap 所有 takeaways；底部 glossary 收录全部术语。
9. **下一步** — 给可执行的继续学习路径。

## 🧱 深度铁律（教学模式的 P0）

① **每个新概念四拍齐**：直觉（一句话/类比）→ 具体例子（挂在贯穿案例上）→ 一句话人话定义 → 检查（一个小问题/揭晓）。缺一拍就是"没教透"。

② **全篇围绕一个贯穿例子**：选一个零基础也有共鸣、且能自然长出多个概念的例子，从头跟到尾、反复复现（用 `running-example` 视觉锚点标记）。**反例**：像现有 `ml-tutorial.html` 那样猫/邮件/鸢尾花各提一句就丢——学习者脑子里建不起一个可咀嚼的场景。
   - ML 推荐例子：「复习时长 → 考试分数」（线性回归直觉，自然带出 特征/标签/训练/预测/过拟合）。

③ **术语先定义后用**：禁止前向引用还没解释的词。需要用到某词，要么已经定义过，要么当场用 `term-card` 定义。

④ **必须回答"它怎么work的"**：这是初学者最大的问号。别停在静态结论（"找一条线"），要展示动态过程（预测→看误差→调整→重复）。优先用可点击的 `training-loop`。

⑤ **每大节末有小结，全页底部有术语表**。

⑥ **有主动回忆**：至少一处 mini-quiz 或 reveal-q。纯阅读留存低；让学习者自己先想、再揭晓。

| 组件 | 用途 |

| `term-card` 概念定义卡 | 教一个术语（术语+人话定义+一个例子，三色编码） |

## 🧸 IP 贴纸（老师全程在场）

教学页最适合让 YC 当"陪读老师"。**目标：每个关键 section 都有一个 YC IP 形象出现**，让人感觉是 YC 在亲自讲这一段——这是教学模式比科普页更进一步的地方（科普页克制到每 1-2 屏 1 张即可；教学页几乎每节都有）。

**铁律（见 `SKILL.md`「每个位置按内容生成」）**：每节那张图要**按这节在讲什么单独生成**，动作/道具呼应内容（讲对比就指着两边、讲训练就摇训练机、讲提问就举手）。**禁止全篇复用同一张老贴纸**。无 `image_gen` 时才用现成贴纸占位。

**落地约定**：给每个 section 的 IP 位置留一个 `data-ip-slot="这张该画成什么"` 说明，或集中成一个 `IP_SLOTS` 清单（参考 `examples/ml-teaching.html` 底部）。占位先用 `assets/stickers/` 现成图，能生图时按槽位说明逐个换新图。

仍守：正放不歪、drop-shadow 浮起、移动端缩小但不挡正文。贴纸要够大、贴近标题，别缩在角落显得没存在感。

### 两档 IP：贴纸 vs Simple 解释图（重要）

教学页用**两个层级**的 YC IP，别只用一种：

| | 轻贴纸（每节都有） | Simple 模式解释图（全页仅一张） |
|---|---|---|
| 形态 | 单角色透明贴纸，"老师在场" | yc-ip Simple/Standard：YC 操作一个低科技装置/场景，把抽象机制演出来 |
| 数量 | 每个 section 一张 | **整页只用 1 张**，给那个**最抽象/最 key 的概念** |
| 位置 | 标题旁，中等大小 | **居中**，当那一节的主视觉，最 standout |
| 例（ML 页） | 各节的姿势占位 | 「训练循环」那节：YC 操作训练机 试做→反馈→调整→再跑（`assets/illustrations/`） |

**为什么只用一张**：Simple 解释图信息量大、视觉重。给最难的那个概念用一张当点睛，能真正帮理解；**每节都放则全页过重、喧宾夺主**。其余概念用轻贴纸 + 自制 SVG/组件即可。

实际生成时：这张 Simple 解释图也要按页面的贯穿例子来画（如 ML 页就画「复习时长→分数」那条线的训练机），不要直接套用别的主题的现成图——现成的只作占位。
